# 房屋租赁AI营销系统 PRD

## 1. 产品目标

打造一款面向房产中介的AI营销助手，通过上传房源图片自动生成吸引人的营销文案，并支持一键发布到小红书和微信公众号，显著降低中介的内容创作成本，提升获客效率。

## 2. 用户故事

1. **作为房产中介**，我想要上传房源图片后自动生成营销文案，以便快速产出高质量推广内容，节省文案撰写时间。

2. **作为房产中介**，我想要在发布前预览和编辑AI生成的文案，以便根据实际情况调整内容，确保信息准确且符合个人风格。

3. **作为房产中介**，我想要一键将文案发布到小红书和微信公众号，以便扩大曝光渠道，提升房源的触达率。

4. **作为房产中介**，我想要系统能根据房源特点自动生成合适的标签和标题，以便提高内容在平台上的搜索可见性。

## 3. 需求池

| 优先级 | 需求 | 说明 |
|--------|------|------|
| **P0** | 房源图片上传 | 支持多图上传，格式支持jpg/png，单图≤10MB |
| **P0** | AI文案生成 | 调用DeepSeek API，生成标题+正文+标签，响应时间<10秒 |
| **P0** | 文案预览与编辑 | 提供富文本编辑器，支持实时修改AI生成内容 |
| **P0** | 小红书发布集成 | 通过xiaohongshu-mcp实现一键发布，支持图片+文案同步 |
| **P1** | 微信公众号发布 | 对接微信官方API，支持图文消息发布 |
| **P1** | 历史记录管理 | 保存生成历史和发布记录，支持查看/删除/复用 |
| **P1** | 文案模板配置 | 允许用户选择不同风格模板（如：专业型、亲和型、紧迫型） |
| **P2** | 批量生成功能 | 支持一次上传多套房源，批量生成文案 |
| **P2** | 数据分析面板 | 展示各平台发布内容的阅读量、互动数据 |
| **P2** | 多账号管理 | 支持绑定多个小红书/公众号账号，灵活选择发布目标 |

## 4. 核心链路说明

```
上传房源图片 
    ↓
AI生成文案（标题+正文+标签）
    ↓
预览/编辑（富文本编辑器）
    ↓
选择平台发布（小红书/公众号）
    ↓
发布成功反馈
```

**详细流程：**
1. 用户拖拽/选择房源图片上传
2. 系统调用DeepSeek API分析图片内容（户型、装修、亮点）
3. AI生成营销文案（标题建议、正文描述、热门标签）
4. 用户在前端预览完整内容，可手动编辑调整
5. 选择发布平台（可多选），点击一键发布
6. 系统调用对应平台API完成发布，返回发布链接/状态

## 5. 技术约束

- **后端框架**：FastAPI，端口 **8899**（统一）
- **前端框架**：Vite + React + Vant UI，端口 **3000**（统一）
- **API路径前缀**：`/api/v1/`（全平台统一）
- **CORS配置**：后端允许 `http://localhost:3000`（开发环境）
- **AI模型**：DeepSeek API（OpenAI兼容格式）
- **数据库**：SQLite（SQLAlchemy async）
- **小红书集成**：xiaohongshu-mcp（HTTP API，端口18060）
- **公众号集成**：微信官方API（需公众号权限）

## 6. 待确认问题

1. **小红书发布的登录态管理**：xiaohongshu-mcp是否需要用户提前在浏览器登录小红书？还是支持cookie注入？这影响前端是否需要增加登录引导流程。

2. **微信公众号的发布权限**：是需要服务商号（有发布接口）还是订阅号（只能群发）？这决定了功能范围和用户群体定位。

3. **AI文案生成的计费策略**：DeepSeek API按token计费，是否需要限制用户每日生成次数？还是完全由用户自己的API key控制？

4. **图片存储方案**：用户上传的房源图片是临时存储（生成后删除）还是长期保存（方便历史复用）？这影响存储成本和隐私合规。

5. **发布失败的重试机制**：如果小红书/公众号API调用失败，系统应该如何提示用户？是否需要草稿箱功能保存未完成发布的内容？
